# UniPin Game-TopUp System

This is the final project submission for **FESE306 – Software Modeling & Design**.
It contains a complete Software Design Document (SDD) with UML models, layered architecture, and a working prototype demonstrating 6 software design patterns.

## Project Structure
- `Final_UniPin_System_SDD.md`: The complete, 80-point Software Design Document containing all Use Cases, Class Diagrams, Sequence Diagrams, and Pattern explanations.
- `UniPin_API.postman_collection.json`: A Postman collection to test the backend API independently.
- `backend/`: Node.js/Express server acting as the Facade to Game Publishers and managing Order State.
- `frontend/`: React/Vite UI implementing Singleton, Builder, Abstract Factory, and Strategy patterns.

---

## How to Run the Prototype (20 Bonus Points)

The prototype demonstrates observable behaviors for the design patterns. It requires Node.js to be installed.

### 1. Start the Backend
Open a terminal and run:
```bash
cd backend
npm install
node src/index.js
```
*The backend runs on `http://localhost:3001`.*

### 2. Start the Frontend
Open a second terminal and run:
```bash
cd frontend
npm install
npm run dev
```
*The frontend will run on `http://localhost:5173`. Open this URL in your browser.*

### 3. Demo Steps for Presentation
1. **Browse Games**: Select a game from the home page (loads from `GameCatalog` Singleton).
2. **Verify Player**: Enter ID `12345` to see a successful verification (routed through `PublisherFacade`).
3. **Select Package**: Pick any package.
4. **Apply Promo**: Enter `DISCOUNT10` or `HALFOFF` to see the price update dynamically.
5. **Checkout**: Select ABA or ACLEDA and click "Pay Now" (executes `PaymentGatewayFactory` and `OrderBuilder`).
6. **Receipt**: You will see a custom receipt generated by the specific bank's factory, and the backend logs will show the `OrderState` transition from Pending -> Paid -> Completed.

## Design Patterns Mapping (For Q&A Defense)

During your presentation, use this guide to show exactly where the patterns are in the code:

### Creational
1. **Singleton**: `frontend/src/patterns/GameCatalog.ts` - Ensures only one instance of the game data is loaded.
2. **Builder**: `frontend/src/patterns/OrderBuilder.ts` - Builds the complex `TopUpOrder` object step-by-step.
3. **Abstract Factory**: `frontend/src/patterns/PaymentGatewayFactory.ts` - Creates families of payment strategies and receipt generators for ABA vs ACLEDA.

### Structural
4. **Facade**: `backend/src/patterns/PublisherFacade.js` - Hides the complexity of Moonton/Tencent APIs and provides simple `validatePlayer` and `deliverCurrency` methods to the core system.

### Behavioral
5. **Strategy**: `frontend/src/patterns/PaymentStrategy.ts` - Allows the system to swap payment algorithms (ABA, ACLEDA, E-Wallet) dynamically without `if/else` chains in the main processor.
6. **State**: `backend/src/patterns/OrderState.js` - Manages the strict order lifecycle (`Pending`, `Paid`, `Completed`, `Failed`) using a finite state machine pattern.
